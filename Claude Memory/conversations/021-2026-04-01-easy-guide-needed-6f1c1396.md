 Here's a simple guide based on the video:

---

**Run Claude Code for Free via OpenRouter**

**What you need:**
- Claude Code installed
- A free OpenRouter account + API key (openrouter.ai)

**Step 1 — Get your API key**
Sign up at openrouter.ai, go to API Keys, create a key, and copy it.

**Step 2 — Find a free model**
Go to the OpenRouter models page and search "free". The recommended option is **openrouter/auto** (the free router at the bottom of the list) — it routes your request to whichever free model is available, so you avoid rate limit errors.

**Step 3 — Configure Claude Code**

For a **single project**, create a `.claude/settings.json` file in your project folder.

For **all projects globally**, find your global Claude config folder and edit the `settings.json` there.

Either way, add these variables:

```json
{
  "env": {
    "OPENROUTER_API_KEY": "your-key-here",
    "ANTHROPIC_BASE_URL": "https://openrouter.ai/api/v1",
    "ANTHROPIC_AUTH_TOKEN": "your-key-here",
    "ANTHROPIC_API_KEY": "",
    "CLAUDE_MODEL": "openrouter/auto"
  }
}
```

**Step 4 — Relaunch Claude Code**
You should see `openrouter/auto` (or your chosen model) as the active model.

---

**Tips:**
- You can swap in any model name (e.g. `qwen/qwen3-coder`) just by changing the model value
- Free models work but aren't as capable as Sonnet/Opus
- If hitting rate limits, try `openrouter/auto` instead of a specific model

 This is the OpenRouter SDK example for sending a **multimodal streaming request** (text + image + audio + video). Here's a quick breakdown:

---

**What it does:**

1. Creates an OpenRouter client with your API key
2. Sends a streaming chat request to `openrouter/auto` with mixed media content
3. Streams the response chunk by chunk and prints it

---

**The message content types:**

| Type | What it is |
|------|-----------|
| `text` | Plain text question |
| `image_url` | Image via URL |
| `input_audio` | Base64-encoded audio (WAV) |
| `video_url` | Video via URL |

---

**To use this yourself:**

1. Install the SDK:
```bash
npm install @openrouter/sdk
```

2. Replace the API key:
```js
apiKey: "sk-or-v1-your-key-here"
```

3. Swap in your own model if needed — not all free models support audio/video, so you may need a paid one for full multimodal support.

---

**Things to watch out for:**

- The `input_audio.data` must be a **complete** base64 string — the example above is truncated and won't work as-is
- `openrouter/auto` will pick a model, but it may not support all modalities — check the model's capabilities on the OpenRouter models page if you get errors
- Streaming works best in Node.js 18+
