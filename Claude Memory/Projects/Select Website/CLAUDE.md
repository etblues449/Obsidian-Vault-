## Working rules

- Pure static site: HTML + CSS + minimal vanilla JS. No build step unless
  asked.
- Keep layouts mobile-first; phone is the primary preview target.
- When proposing changes, point at the corresponding lines in the
  Select-lifestyles-Website- repo so the user can review the diff there.
- Brand voice: lifestyle / wellness — clean, warm, not corporate-stiff.
- Image assets stay in the repo; do not pull from external CDNs unless
  explicitly requested.
